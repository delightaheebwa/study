BPE Tokenizer from Scratch

Motto: A tokenizer is the bridge between raw text and the numbers a neural network can process. Build it once, and you'll never treat tokenizers as magic again.

The Problem

You want a language model to process the sentence: "The lower the learning rate, the lower the loss."

A neural network can't eat characters or words — it eats numbers. Each token needs a unique integer ID. But how do you decide what counts as a "token"?

  • Word-level: ["The", "lower", "the", "learning", "rate", "the", "loss"] — vocabulary explodes (~170k words in English). Misspellings and new words get [UNK].
  • Character-level: ["T","h","e"," ","l","o","w","e","r",...] — tiny vocab (26+), but sequences get 10x longer, making it harder for the model to learn long-range patterns.

The tradeoff: Big tokens (words) → short sequences but huge vocabulary. Small tokens (characters) → tiny vocabulary but long sequences. We need something in between.

The Concept: Byte-Pair Encoding

BPE starts with characters and iteratively merges the most frequent pair of adjacent tokens. The result is a subword vocabulary — common words stay whole, rare words are split into familiar pieces.

flowchart LR A["c h a r a c t e r"] --> B["c h ar a c t e r"] B --> C["ch ar a c t e r"] C --> D["ch ar a c te r"] D --> E["ch ar ac te r"] E --> F["ch ar ac te r"] F --> G["char ac ter"] G --> H["character"]

Figure: BPE merges the most frequent pair at each step. "character" emerges from c + h → ch, a + r → ar, t + e → te, etc.

Build It: BPE from Scratch

Let's implement BPE in ~60 lines of Python. We'll train on a tiny corpus so you can see every merge.

Step 1: Pre-tokenize (split into words with spaces)

corpus = [
    "the lower the learning rate the lower the loss",
    "learn to lower the learning rate",
    "the lowest loss is the goal"
]

def pre_tokenize(text):
    """Split on whitespace and add word-boundary marker ''."""
    return [list(word) + [""] for word in text.split()]

pretokenized = [pre_tokenize(s) for s in corpus]
# [['t','h','e',''], ['l','o','w','e','r',''], ...]

Step 2: Count pair frequencies

from collections import Counter

def get_pair_counts(pretokenized):
    """Count every adjacent pair of tokens across the corpus."""
    counts = Counter()
    for sentence in pretokenized:
        for word in sentence:
            for i in range(len(word) - 1):
                counts[(word[i], word[i+1])] += 1
    return counts

pair_counts = get_pair_counts(pretokenized)
# Most frequent pair: ('l', 'o') appears how many times?

Quick Check

In the corpus above, which adjacent pair appears most frequently?

('t', 'h') — "th" appears in "the" (3x) ('l', 'o') — "lo" appears in "lower" + "lowest" + "loss" = 5 occurrences ('e', 'r') — "er" appears in "lower" + "learning" ('a', 'r') — "ar" appears in "rate" (2x)

Step 3: Merge the best pair

def merge_pair(pretokenized, pair):
    """Merge every occurrence of (a, b) into a new token 'ab'."""
    a, b = pair
    merged = []
    for sentence in pretokenized:
        new_sentence = []
        for word in sentence:
            new_word = []
            i = 0
            while i < len(word):
                if i < len(word) - 1 and word[i] == a and word[i+1] == b:
                    new_word.append(a + b)
                    i += 2
                else:
                    new_word.append(word[i])
                    i += 1
            new_sentence.append(new_word)
        merged.append(new_sentence)
    return merged

Step 4: The full BPE training loop

def train_bpe(corpus, vocab_size=20):
    pretokenized = [pre_tokenize(s) for s in corpus]
    vocab = set()
    merges = []

    # Collect initial characters
    for sentence in pretokenized:
        for word in sentence:
            for char in word:
                vocab.add(char)

    while len(vocab) < vocab_size:
        counts = get_pair_counts(pretokenized)
        if not counts:
            break
        best_pair = counts.most_common(1)[0][0]
        merges.append(best_pair)
        pretokenized = merge_pair(pretokenized, best_pair)
        new_token = best_pair[0] + best_pair[1]
        vocab.add(new_token)
        print(f"Merge {len(merges)}: {best_pair} → '{new_token}' (freq: {counts[best_pair]})")

    return merges, pretokenized

merges, tokenized = train_bpe(corpus, vocab_size=30)

Step 5: Encode new text using the learned merges

def encode(text, merges):
    """Apply learned merges to tokenize new text."""
    words = pre_tokenize(text)
    for word in words:
        while True:
            stats = Counter()
            for i in range(len(word) - 1):
                stats[(word[i], word[i+1])] = 1
            # Find the first merge in our list that applies
            best = None
            for pair in merges:
                if pair in stats:
                    best = pair
                    break
            if best is None:
                break
            # Apply merge
            a, b = best
            new_word = []
            i = 0
            while i < len(word):
                if i < len(word) - 1 and word[i] == a and word[i+1] == b:
                    new_word.append(a + b)
                    i += 2
                else:
                    new_word.append(word[i])
                    i += 1
            word = new_word
    return word

tokens = encode("lowest loss", merges)
print(tokens)  # ['low', 'est', '', 'lo', 'ss', '']
Try it: Run this in a Colab notebook or local Python. Print the merge sequence — you'll see how "lo" + "wer" → "lower" emerges naturally from raw frequency statistics.

Step 6: Build the token-to-ID mapping

def build_vocab(merges, special_tokens=["[PAD]", "[UNK]", "[CLS]", "[SEP]"]):
    """Assign integer IDs to every token in the vocabulary."""
    tokens = list(special_tokens)
    seen = set(tokens)
    for a, b in merges:
        t = a + b
        if t not in seen:
            tokens.append(t)
            seen.add(t)
    return {t: i for i, t in enumerate(tokens)}

vocab = build_vocab(merges)
print(vocab)
# {'[PAD]': 0, '[UNK]': 1, '[CLS]': 2, '[SEP]': 3, 't': 4, 'h': 5, ...}

Use It: HuggingFace Tokenizers Library

Now that you understand how BPE works internally, let's use the production-grade library:

pip install tokenizers

from tokenizers import Tokenizer, models, trainers, pre_tokenizers

# BPE model
tokenizer = Tokenizer(models.BPE())
tokenizer.pre_tokenizer = pre_tokenizers.ByteLevel(add_prefix_space=True)

trainer = trainers.BpeTrainer(vocab_size=30000, special_tokens=["[PAD]", "[UNK]", "[CLS]", "[SEP]"])
tokenizer.train_from_iterator(corpus, trainer)

output = tokenizer.encode("the lower the learning rate")
print(output.tokens)
# ['the', 'Ġlower', 'Ġthe', 'Ġlearning', 'Ġrate']
print(output.ids)
# [1234, 5678, 1234, 9101, 1121]

What changed? The library version adds a Ġ prefix to indicate a space-preceded token (so "lower" mid-sentence differs from "Lower" capitalized). Your from-scratch version used </w>. Same idea, different convention.

Ship It: Your BPE Skill

You now understand tokenization deeply. Here's a reusable skill definition for your agent:

---
name: bpe-tokenizer
description: Train and use a BPE tokenizer from scratch
phase: 10
lesson: 01
---

Given a corpus and a desired vocabulary size, train a BPE tokenizer:
1. Pre-tokenize by splitting on whitespace, marking word boundaries
2. Count adjacent pair frequencies across the corpus
3. Merge the most frequent pair
4. Repeat until vocabulary reaches target size
5. Store the merge-ordered list for encoding new text
6. Assign integer IDs to each token in the vocabulary, reserving special tokens for [PAD], [UNK], [CLS], [SEP]

Final Check

After training BPE on "low low lower", what's the first merge?

('l', 'o') — appears in "low" (3x) ('o', 'w') — appears in "low" + "low" + "low" = 3x — wait it's the same as ('l','o')... ('l', 'o') and ('o', 'w') both appear 3 times. The tiebreaker is arbitrary — either is correct.

Key Takeaways

  • BPE finds the optimal subword vocabulary for your corpus via frequency statistics
  • Tokenization is lossless — you can reconstruct the original text from token IDs
  • Special tokens must be reserved before training so their IDs don't conflict
  • Modern tokenizers (GPT-4, Claude, Llama 3) use variants like SentencePiece or Unigram, but they share the same core idea: subword merging
